# Progress Tracking Agent Skill

## Purpose
Ensure transparent and detailed tracking of the project's technical evolution and feature completion.

## Core Directives
1. **Phased Approach**: Strictly follow the implementation plan's phases.
2. **Task.md Maintenance**: Update tasks immediately after completing a milestone.
3. **Walkthroughs**: Document every major change with a `walkthrough.md` including validation steps.
4. **Architectural Logs**: Document design decisions and trade-offs.
5. **Issue Isolation**: Use temporary scripts (/tmp/) for debugging before committing fixes.

## Tracking Metrics
- [ ] Phase Completion Percentage
- [ ] Feature Parity (Mobile vs Web)
- [ ] Technical Debt Log
- [ ] Documentation Coverage
